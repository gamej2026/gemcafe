### CR-004 · 결과 판정 (코어/서브 3단계) · MUST
- 근거: 0807 p.25 "결과 판정".
- 규칙(맛 기준 비교):
  - 레시피는 **코어 맛 1개 + 서브 맛 2개**로 정의.
  - 플레이어는 재료 3개 → 맛 3개를 제출.
  - 판정:
    1. 제출 맛에 **코어 맛이 없으면 → 실패**.
    2. 코어 맛 포함 시, 나머지에서 **서브 2개 중 일치 개수**로:
       - 2개 일치 → **대성공**
       - 1개 일치 → **성공**
       - 0개 일치 → **성공** `[TBD]`(기획 예시 미정의, 기본=성공: 코어가 핵심)
- 예시 검증(p.25, 정답 코어=단맛 / 서브=신맛,매운맛):
  - 단맛,매운맛,짠맛 → 성공 · 매운맛,신맛,짠맛 → 실패 · 단맛,신맛,감칠맛 → 성공 · 단맛,신맛,매운맛 → 대성공.
- AC:
  - [ ] 위 4개 예시가 각각 성공/실패/성공/대성공으로 판정된다(유닛테스트).
  - [ ] 코어 불일치는 무조건 실패다.
  - [ ] 판정 결과는 `enum DrinkResult { Fail, Success, GreatSuccess }`로 반환된다.
- 구현 메모: [RecipeEvaluator.cs](../Assets/_Game/Scripts/Crafting/RecipeEvaluator.cs)의 `bool Matches(...)`를 `DrinkResult Evaluate(bowlTastes, recipe)`로 교체. 입력은 재료→맛 매핑(§4 CR-013).
- [TBD] Q-B: 동일 맛 중복 제출 허용/판정 방식(집합 기준 vs 멀티셋). 기본=**집합 기준(중복 무시)**.

### CR-005 · 코인 보상 모델 · MUST
- 근거: 0807 p.29 "코인을 얻었다", p.31~ 엔딩이 코인 수/등급 기반.
- 동작:
  - 결과별 코인:
    - 대성공 → **대성공 코인 1개** 획득(총 코인 +1, 대성공 코인 +1).

    - 실패 → **획득 없음**.
  - 누적값: `totalCoins`, `greatCoins`(대성공 코인 수).
- AC:
  - [ ] 결과 등급에 따라 코인이 0/1개 가산된다.
