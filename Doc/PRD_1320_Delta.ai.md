## 2. 갱신된 제조 서브 상태머신

기존 `ServiceSubState`(None/CustomerEnter/OrderDialogue/Crafting/FinishAnim/Result)의
`Crafting` 단계를 아래 하위 단계로 세분화한다.

```
CRAFTING
  ├─ INGREDIENT_SELECT   (재료 서랍 → 막자사발 D&D, 막자 클릭 대기)
  │     └─(막자 클릭)→ MIX_MINIGAME
  ├─ MIX_MINIGAME        (막자사발 섞기 미니게임: 진척도 0~100%)
  │     ├─ 성공(100%)─→ POUR_PREP
  │     └─ 실패(0%)  ─→ (재시도 or 실패 누적 — §FR-031 [TBD])
  ├─ POUR_PREP           (다기 클릭 → 막자사발 기울임 + 뜨거운 물 연출 → 차 따르기 씬 전환)
  ├─ POUR_MINIGAME       (차 따르기 미니게임: 수위 75~95%)
  │     ├─ 성공─→ DRINK_COMPLETE
  │     └─ 실패─→ (§FR-031 [TBD])
  └─ DRINK_COMPLETE      (반짝 + 음료 이름 → 손님 앞 건네기)
        └─→ RESULT (기존 흐름)
```

규칙:
- `INGREDIENT_SELECT`/`POUR_PREP`에서 카메라는 **차 만들기 구도**(재료 서랍 전면).
- `POUR_MINIGAME`/`DRINK_COMPLETE`/`RESULT`에서는 **손님 접대 구도**(손님 중앙).
- 인내심 타이머는 폐기 상태이므로 본 플로우에서 시간 제한을 두지 않는다(`[ASSUMPTION]`).

권장 구현: 기존 `ServiceSubState`에 enum 값을 추가하지 않고, `Crafting` 내부 단계는
`CraftingController`가 보유한 별도 `CraftStage` enum으로 관리한다(상위 상태머신 오염 방지).

### 3.1 화면 구도 / 듀얼 뷰

**FR-024 · 손님 접대 ↔ 차 만들기 듀얼 뷰 전환 · MUST · `[CHANGED]`**
- 대상: 기존 `ScreenTransition`(우→좌 단순 슬라이드)을 듀얼 뷰 전환기로 확장.
- 동작:

  - 시선(카메라) 또는 오브젝트 묶음이 **좌우로 보간 이동**해 두 구도를 전환.
  - 손님 접대 구도: 손님이 화면 중앙, 대화창은 손님 아래.
  - 차 만들기 구도: 재료 서랍이 화면 전면, 손님은 화면 기준 좌측에 작게 보임.
- AC:
  - [ ] 제조 진입 시 손님 접대 → 차 만들기 구도로 부드럽게 이동(보간, 인스펙터 노출 duration).
  - [ ] 차 따르기 단계 진입 시 다시 손님 접대 구도로 복귀.
  - [ ] UI 3버튼(레시피 참고 / 대화 다시 보기 / 설정)은 구도 이동과 **무관하게 화면 우상단 고정**.
  - [ ] 「얻은 코인 표기」 HUD도 화면 고정(이동에 영향받지 않음).
- 구현: 카메라 이동 또는 루트 오브젝트 anchoredPosition 보간 중 택1(`[ASSUMPTION]` 2D URP이므로 루트 오브젝트 이동 권장).

### 3.3 막자사발 섞기 미니게임

**FR-027 · 섞기 미니게임 (스타듀밸리 낚시류) · MUST · `[NEW]`**
- 진입: FR-026의 막자 클릭. 막자사발이 화면에서 확대되고, **좌측에 세로 진척도 게이지** 생성.
- UI 요소:
  - 채집 진척도 게이지(우측 또는 좌측 세로 바): 0~100%.
  - 채집 구역 바(Bar): 플레이어가 입력으로 높낮이를 조작하는 녹색 영역.
  - 찻잎 아이콘(Target): 채집 구역 바 영역 내부에서 고유 패턴으로 상하 이동.
- 조작(Hold & Release):
  - 버튼/마우스 Hold → 채집 구역 바 상승.
  - Release → 중력에 의해 하강.
  - 연타로 미세 조정 가능.
- 규칙:
  - 시작 진척도 = 40%(인스펙터).
  - 채집 구역 바 안에 찻잎이 겹쳐 있는 동안 진척도 **상승**.
  - 찻잎이 바를 벗어나면 진척도 **하락**.
  - 100% 도달 → **섞기 성공** → `POUR_PREP`.
  - 0% 도달 → **섞기 실패** → §FR-031 실패 처리 분기.
- AC:
  - [ ] 시작 시 게이지 40%에서 시작.
  - [ ] Hold 시 바 상승, Release 시 하강(가속도/중력 인스펙터 노출).
  - [ ] 찻잎이 바 내부일 때 진척도 증가, 외부일 때 감소(증감 속도 인스펙터 노출).
  - [ ] 찻잎 이동 패턴은 데이터(곡선/속도/난이도)로 분리(`[ASSUMPTION]` 손님/일자별 난이도 조정 여지).
  - [ ] 100%/0% 도달 시 각각 성공/실패 콜백 발생, 중복 발화 방지.
- 데이터: `MixMinigameConfig`(startProgress, riseSpeed, gravity, progressGainRate, progressLossRate, leafPattern) SO 또는 직렬화 구조체.

**FR-028 · 섞기 → 사발 담기 & 뜨거운 물 연출 · MUST · `[NEW]`**
- 진입: FR-027 섞기 성공 직후.
- 동작:
  - **다기(Teaware) 클릭** 대기 상태로 전환(가이드 표시 권장).
  - 다기 클릭 시: 막자사발이 기울어져 내용물이 다기에 담기는 연출 → 그 위로 뜨거운 물이 부어지는 연출.
  - 연출 종료 후 차 따르기 씬(손님 접대 구도)으로 전환.
- AC:
  - [ ] 섞기 성공 전에는 다기 클릭이 비활성.
  - [ ] 다기 클릭 → 기울임 + 물붓기 연출 재생(연출 시간 인스펙터 노출).
  - [ ] 연출 완료 후 `POUR_MINIGAME` 진입 + 카메라 손님 접대 구도 복귀(FR-024 연동).

### 3.4 차 따르기 미니게임

**FR-029 · 차 따르기 미니게임 (수위 맞추기) · MUST · `[NEW]`**

  - 목표 범위: 수위 **75%~95%**(인스펙터 노출, min/max).

  - 정지 시(또는 시간 종료 시) 수위가 75~95% → **따르기 성공**.
  - 75% 미만(모자람) 또는 95% 초과(넘침) → **따르기 실패**.
- AC:
  - [ ] Hold 동안 수위가 0%에서부터 연속 상승(상승 속도 인스펙터 노출).
  - [ ] 75~95% 구간이 시각적으로 구분(가이드 마커/색).
  - [ ] 확정(Release 후 일정 시간 또는 확정 입력) 시 범위 판정 → 성공/실패 콜백.
  - [ ] 넘침(>100%) 시 즉시 실패 처리(`[ASSUMPTION]`).
- 데이터: `PourMinigameConfig`(fillSpeed, targetMin=0.75, targetMax=0.95, overflowFail=true).

### 3.6 최종 판정 — 재료 + 미니게임 연동

**FR-031 · 미니게임 성패와 결과 판정 합산 · MUST · `[NEW]`**
- 배경: 1320은 재료 판정 외에 섞기·따르기 미니게임의 성패가 추가됨. 결합 규칙이 기획서에 명시되지 않아 아래 기본 규칙을 채택(`[ASSUMPTION]`, 구현 전 확정 권장).
- 입력: `recipeResult ∈ {GreatSuccess, Success, Fail}`(기존 `RecipeEvaluator.Evaluate`), `mixOk ∈ {성공, 실패}`, `pourOk ∈ {성공, 실패}`.
- 기본 결합 규칙(`[ASSUMPTION]`):
  - 섞기 실패(0% 도달) → 최종 **실패**(Fail), 이후 단계 스킵하고 RESULT(Fail)로 직행.
  - 따르기 실패 → 최종 **실패**(Fail).
  - 두 미니게임 모두 성공 → 최종 결과 = `recipeResult`(대성공/성공/실패 그대로).

- AC:
  - [ ] 섞기/따르기 실패는 `EventBus.RaiseDrinkResult(DrinkResult.Fail)` 경로로 RESULT 진입.
  - [ ] 미니게임 전부 성공 시에만 `RecipeEvaluator.Evaluate` 결과를 그대로 사용.
