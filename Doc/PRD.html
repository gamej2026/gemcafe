1. 문서 개요

본 문서는 첨부 기획서 「젬카페 UI」(21페이지)를 구현 가능한 제품 요구사항으로 정리한 PRD이다. 게임의 흐름, 화면별 UI/UX, 핵심 시스템, 데이터 모델, 미결 사항을 포함한다.

대상 독자

  • 기획 — 의도/규칙 합의 및 미결 항목 확정
  • 클라이언트 개발 — 화면/시스템 구현 범위 파악
  • 아트/사운드 — 필요 에셋(캐릭터, 배경, 효과음 등) 식별

용어

  • 삼도천 — 이승과 저승 사이의 강. 게임의 시작 배경.
  • 인내심(Patience) — 손님별 제한시간 역할의 게이지.
  • 사발 / 막자 — 재료를 담는 그릇 / 빻고 섞는 막대 도구.
  • 트레이 — 재료를 보관·선택하는 패널.

우선순위 표기: MUST 필수 SHOULD 권장 COULD 선택

2. 게임 개요 & 목표

컨셉

기억을 잃은 주인공이 삼도천 앞에서 깨어나, 강을 건널 뱃삯이 없어 강변 카페에 취업한다. 배 출발까지 3일 동안 손님에게 음료를 제조해 주며 뱃삯을 모으는 것이 목표.

장르 / 핵심 루프

비주얼노벨 + 드래그&드롭 제조 시뮬레이션.

손님 등장→ 주문 대화→ 재료 제조→ 막자로 믹스→ 평가

플레이어 목표 / 실패 조건

  • 목표 — 3일간 손님 주문을 정확히 제조하여 뱃삯 확보.
  • 긴장 요소 — 손님별 인내심(제한시간)과 목숨 3개.
  • 실패 — 인내심이 모두 소진되면 해당 손님 실패 처리(목숨 차감).

3. 전체 게임 흐름

로비→ 인트로 스토리→ S1 카페로 가는 길
(이동·대화)→ 카페 진입
(취업)→ 튜토리얼→ 본게임
(손님·제조)→ 완료 애니메이션

참고 본게임은 손님 단위로 반복되며, 3일(일자 단위)로 묶인다. 일자/진행도 관리 방식은 9장 미정 사항 참고.

4. 내러티브 / 스토리

4.1 인트로

  1. 주인공이 기억을 잃은 채 삼도천 앞에서 깨어남.
  2. 삼도천을 따라 걷다 강을 바라보며 — "맨몸으로 건너고 싶지 않아…"
  3. 멀리서 돌쇠가 부름 → 돌쇠 방향으로 이동.
  4. 돌쇠와 대화 — "배 타고 싶으면 돈 줘."
  5. 대화 중 마님이 카페에서 나오며 끼어듦 — "야! 돌쇠가 먼저 말을 걸었다고? 일단 이리 와봐."

4.2 카페 진입

  • 마님과 대화 후 카페에 취업.
  • 독백 — "전생에 무직백수 청년이던 내가 지옥에선 한방에 취업?" · "잠시 잊고 있던 기억이 떠오른 것 같다…"
  • 동기 부여 — "배 출발까지 3일, 뱃삯을 벌어보자!"
  • 수락 대사 — "까짓꺼 한번 해보죠."

5. 화면별 상세 명세

5.1 로비

구성 요소

  • 상단 중앙: 게임 이름 타이틀
  • 중앙 메뉴: 새 게임 / 이어하기 / 설정 / 게임 종료

동작

  • 새 게임 → 인트로 스토리 진입
  • 이어하기 → 저장된 진행도 로드
  • 설정 → 설정 팝업
  • 게임 종료 → 애플리케이션 종료
┌────────────────────────────┐ │ [ 게임 이름 ] │ │ │ │ ┌──────────────┐ │ │ │ 새 게임 │ │ │ │ 이어하기 │ │ │ │ 설정 │ │ │ │ 게임 종료 │ │ │ └──────────────┘ │ └────────────────────────────┘

5.2 S1 — 카페로 가는 길

이동 & 상호작용

  • 주인공 횡(좌우) 이동 — ←/→ 또는 A/D.
  • 배경에 강(삼도천) 요소 배치.
  • 멀리서 부르는 연출은 말풍선만 표시(대화창 X).
  • NPC 근접 시 NPC 테두리 발광 + 화면 상단에 상호작용 키 안내 표시.
  • 상호작용 키 — F.
┌────────────────────────────┐ │ (멀리서 호출) │ │ 💬 말풍선 │ │ ┌─────┐ ╔═════╗ │ │ │주인공 │ (강) ║ NPC ║◄ F │ │ └─────┘ ╚═════╝ │ │ ←/→ 또는 A/D 이동 │ └────────────────────────────┘

대화 (비주얼노벨)

  • 좌측 주인공, 우측 대화 상대 스탠딩 배치.
  • 화자 강조 — 비화자 측을 약간 어둡게 처리, 배경에 딤(dim).
  • 대화창 우측 하단 「다음」 버튼으로 진행.
  • 끼어들기 연출 — 마님의 말풍선 "야!"를 별도 표시, 말풍선 뒤로 딤 처리.
  • 카메라 이동 — 마님 "너 일로 와봐" 시 카페(오른쪽)로 카메라 이동.
  • 따라가기 — 주인공이 마님을 따라 카페로 들어가는 이동 애니메이션.

5.3 카페 진입 (대화)

카페 내부에서 마님과 대화 → 취업 확정. 화면 구성은 5.2의 비주얼노벨 대화 규칙(화자 강조·딤·다음 버튼)을 동일하게 따른다.

5.4 튜토리얼

선택 사항 튜토리얼을 넣는다면 카페 진입 직후, 본게임 직전 타이밍에 배치. 제조 조작(트레이→사발 드래그, 막자 믹스)을 안내. COULD

5.5 본게임 — 손님 응대 & 음료 제조

A. 손님 등장 & 주문 대화

  • 입장 효과음 "딸랑~" + 손님 이미지 페이드 인.
  • 중앙 대화 상대, 하단 바텐더 테이블.
  • 좌상단 목숨 3개 표시.
  • 대화는 말풍선으로 표현.
┌────────────────────────────┐ │ [♥♥♥] 딸랑~ (페이드인) │ │ ┌────────┐ │ │ │ 대화상대 │ │ │ └────────┘ │ │ ╭──────[ 말풍선 ]──────╮ │ │════════ 바텐더 테이블 ════════│ └────────────────────────────┘

B. 제조 화면 진입

  • 오른쪽에서 화면이 왼쪽으로 슬라이드(←) 되어 제조 화면 등장.
  • 우측: 테이블(완전 탑뷰) → 트레이(←로 열림) + 테이블 위 사발.
  • 좌측 패널 유지: 목숨 3개, 인내심 게이지, 대화 상대.
  • 좌하단 버튼: 대화 다시 보기 · 레시피 참고 · 설정.
┌──────────────┬───────────────┐ │ [♥♥♥] │ │ │ ▭ 인내심 게이지 │ 트레이 (←로 열림) │ │ ┌────────┐ │ │ │ │ 대화상대 │ ├───────────────┤ │ └────────┘ │ │ │ ┌──┐┌──┐┌──┐ │ 테이블 위 사발 │ │ │다 ││레 ││설││ │ │ │시 ││시 ││정││ (완전 탑뷰) │ │ └──┘└──┘└──┘ │ │ └──────────────┴───────────────┘

C. 재료 투입 (드래그 & 드롭)

  • 트레이의 재료를 드래그&드롭으로 사발에 투입.
  • 드래그 중에는 마우스 포인터에 재료가 부착되어 따라다님.
  • 드롭 시 포인터가 사발 영역 안이면 사발에 넣은 것으로 인식.

D. 좌하단 버튼 팝업

  • 버튼 상호작용 시 해당 팝업 열람, 팝업 뒤로 딤 처리.
  • 대화 다시 보기 / 레시피 참고 / 설정 각각의 팝업.
  • 시간(인내심) 규칙 — 6장 인내심 시스템 참고.

E. 제조 완료 (막자 믹스)

  • 재료를 다 넣은 경우, 사발 옆 막자(막대기)를 사발에 드래그&드롭.
  • → 완료 애니메이션으로 전환.

5.6 완료 애니메이션

막자로 사발 내 내용물을 믹스하는 애니메이션 재생 → 음료 완성 → 손님 평가/결과 처리.

미정 완성 결과 판정(정답 레시피 일치 여부)과 보상/뱃삯 가산 로직은 9장 참고.

6.2 대화 (비주얼노벨)

항목규칙

6.3 음료 제조 (드래그 & 드롭)

단계규칙
트레이 열기← 방향 슬라이드로 트레이 노출.
재료 드래그포인터에 재료 부착, 함께 이동.
드롭 판정포인터가 사발 영역 내부면 투입 인식.
믹스막자를 사발에 드롭 → 완료 애니메이션.

6.4 인내심 시스템 (제한시간)

  • 역할 — 제한시간. 손님별로 서로 다른 인내심 보유.
  • 시간이 흐름에 따라 감소.
  • 설정 팝업에서만 시간이 흐르지 않음(일시정지).
  • 대화 다시 보기 / 레시피 참고 팝업에서는 시간이 흐름. 변동 가능
  • 인내심이 모두 소진되면 실패 처리.
팝업별 시간 흐름 ───────────────────────── 설정 ⏸ 정지 대화 다시 보기 ▶ 흐름 (변동 가능) 레시피 참고 ▶ 흐름 (변동 가능) 일반 제조 중 ▶ 흐름

6.5 목숨 시스템

  • 화면 좌상단에 목숨 3개 표시.
  • 실패(인내심 소진 등) 시 차감되는 것으로 추정 — 규칙 확정 필요.
  • 0이 되면 게임오버 처리(추정) — 9장 참고.

9. 미정 / 추가 논의 사항

확정 필요
